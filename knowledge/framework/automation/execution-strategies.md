# Automation Execution Strategies

## Purpose
This split document isolates automation matrices and execution strategies from the parent Automation Framework.

## Source
- Parent specification: [Automation Framework](../automation-framework.md)

## Matrices
Matrices include automation, trigger, condition, action, workflow, scheduler, background job, command, domain event, rule engine, decision engine, projection, and notification mapping.

## Strategies
Execution strategies include retry, compensation, escalation, approval, failure recovery, idempotency, dependency handling, and notification coordination.
