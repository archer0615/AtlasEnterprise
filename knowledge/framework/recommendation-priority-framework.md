# Recommendation Priority Framework

Version: 1.0

## Purpose
Defines the enterprise framework used by Project Atlas to prioritize, rank, merge, suppress, and explain financial recommendations generated by all engines.

## Objectives
- Present the highest-value recommendations first.
- Resolve conflicts between competing recommendations.
- Prevent recommendation overload.
- Ensure deterministic and explainable prioritization.

## Recommendation Sources
- Decision Engine
- Cash Flow Engine
- Investment Engine
- Loan Engine
- Retirement Engine
- House Decision Engine
- Scenario Engine
- Financial Health Engine

## Recommendation Categories
### Critical
Immediate action required.

### High
Strong financial benefit within short time horizon.

### Medium
Optimization opportunities.

### Low
Informational improvements.

### Informational
Educational insights only.

## Priority Dimensions
Each recommendation is scored using:
- Financial Impact
- Risk Reduction
- Goal Alignment
- Liquidity Impact
- Cash Flow Impact
- Time Sensitivity
- User Preference
- Confidence Score

## Composite Priority Score
Priority Score =
Weighted sum of:
- Financial Impact
- Risk Impact
- Urgency
- Goal Importance
- Sustainability
- Confidence

Normalized to 0–100.

## Recommendation Lifecycle
Draft
→ Generated
→ Ranked
→ Displayed
→ Acknowledged
→ Accepted / Dismissed
→ Archived

## Conflict Resolution Rules
- Hard constraints override recommendations.
- Critical recommendations suppress lower-priority items.
- Duplicate recommendations are merged.
- Contradictory recommendations require scenario comparison.
- User-dismissed recommendations remain hidden until triggering conditions change.

## Recommendation Bundling
Related recommendations may be grouped:
- Cash Flow Bundle
- Housing Bundle
- Retirement Bundle
- Investment Bundle
- Debt Optimization Bundle

## Business Rules
- Recommendations are reproducible from identical inputs.
- Every recommendation has a stable Recommendation ID.
- Recommendations include expiration and refresh conditions.
- Priority recalculates after significant financial events.

## Phase 2 Executable Specification

### Recommendation Priority Contract

| Field | Requirement |
|---|---|
| Aggregate | RecommendationPriority |
| Identity | priorityEvaluationId |
| Scope | tenantId, recommendationId |
| Inputs | recommendation, evaluationResult, goalContext, riskContext, cashFlowContext, userPreferences |
| Outputs | priorityScore, priorityCategory, conflictResults, suppressionDecision, explainabilityMetadata |
| Determinism | Same inputs and weighting model must produce the same priority score and category. |

### Required Commands

| Command | Purpose |
|---|---|
| CalculateRecommendationPriority | Calculate composite priority score and category. |
| ResolveRecommendationConflicts | Resolve competing or contradictory recommendations. |
| MergeDuplicateRecommendations | Merge recommendations with matching fingerprint. |
| SuppressRecommendation | Hide recommendation according to suppression rules. |
| BundleRecommendations | Group related recommendations into a bundle. |
| ReplayRecommendationPriority | Rebuild priority result from stored inputs and model version. |

### Domain Events

| Event | Trigger |
|---|---|
| RecommendationPriorityCalculated | Priority scoring completes. |
| RecommendationConflictResolved | Conflict resolution completes. |
| DuplicateRecommendationsMerged | Duplicates are merged. |
| RecommendationSuppressed | Recommendation is hidden by rule. |
| RecommendationBundleCreated | Related recommendations are grouped. |
| RecommendationPriorityReplayed | Historical priority is reproduced. |

### Validation Rules

1. Composite score must normalize to 0 through 100.
2. Critical recommendations must record hard-constraint or critical-event evidence.
3. Suppression requires reason, trigger condition, and refresh condition.
4. Conflict resolution must retain all alternative recommendation references.
5. Replay must use original weights, inputs, and priority model version.

### API Contract

| Endpoint | Method | Purpose |
|---|---|---|
| /api/recommendations/{recommendationId}/priority/calculate | POST | Calculate priority. |
| /api/recommendations/conflicts/resolve | POST | Resolve conflicts. |
| /api/recommendations/duplicates/merge | POST | Merge duplicates. |
| /api/recommendations/{recommendationId}/suppress | POST | Suppress recommendation. |
| /api/recommendations/bundles | POST | Bundle related recommendations. |
| /api/recommendations/{recommendationId}/priority/replay | POST | Replay priority calculation. |
