# Scheduler Execution Strategies

## Purpose
This split document isolates scheduler matrices and execution strategies from the parent Scheduler Framework.

## Source
- Parent specification: [Scheduler Framework](../scheduler-framework.md)

## Matrices
Matrices include scheduler, background job, workflow, automation, command, and domain event mapping.

## Strategies
Execution strategies include retry, misfire, catch-up, concurrency, idempotency, dependency coordination, and failure handling.
