# Recommendation Execution Policies and Monitoring

## Purpose
This split document isolates execution policies, execution modes, monitoring, health, metrics, KPIs, forecasts, timelines, logs, and validation rules from the parent Recommendation Execution specification.

## Source
- Parent specification: [Recommendation Execution](../recommendation-execution.md)

## Execution Modes
Execution supports Manual, Automatic, Scheduled, Event Driven, Workflow Driven, Batch, Parallel, Incremental, Simulation, Dry Run, and Emergency Execution modes.

## Execution Policies
Execution policies include Priority Policy, Dependency Policy, Retry Policy, Timeout Policy, Cancellation Policy, Rollback Policy, Recovery Policy, Escalation Policy, Approval Policy, Notification Policy, and Audit Policy.

## Execution Monitoring
Monitoring tracks Execution Progress, Execution Status, Execution Health, Execution Metrics, Execution Errors, Execution Warnings, Execution KPIs, Execution Forecast, Execution Timeline, and Execution Logs.

## Validation and Business Rules
Validation Rules and Business Rules preserve governed execution for existing Recommendations while keeping Domain ownership, Catalog naming, validation, audit, security, lifecycle, and execution evidence intact.
