# Decision Insights Generation and Detection

## Purpose
This split document isolates insight generation architecture, pipeline, categories, detection rules, scoring, ranking, and evidence collection from the parent Decision Insights specification.

## Source
- Parent specification: [Decision Insights](../decision-insights.md)

## Insight Architecture
Insight generation coordinates the Insight Engine, Pattern Detection Engine, Trend Analysis Engine, Anomaly Detection Engine, Recommendation Engine, Forecast Engine, Risk Engine, Opportunity Engine, Scoring Engine, Evidence Engine, Audit Engine, and Caching Layer.

## Generation Pipeline
Pipeline steps include data collection, normalization, aggregation, pattern detection, trend analysis, anomaly detection, opportunity discovery, risk discovery, scoring, ranking, evidence collection, recommendation mapping, and audit logging.

## Detection Rules
Detection includes threshold, pattern, historical comparison, forecast comparison, peer comparison, scenario comparison, rule-based, event-driven, scheduled, and incremental detection.
