## Purpose
This split document isolates API rule catalogs, Mermaid flows, edge cases, and completion matrices from the parent API Governance Framework while preserving the parent document as the canonical full specification.

## Rule Families
The parent framework defines API governance rules for error code behavior, validation handling, business policy enforcement, compatibility checks, resource consistency, and endpoint behavior.

## Flow Families
The parent framework keeps the flow diagrams for REST Architecture, Request Flow, Authentication Flow, Authorization Flow, Error Flow, and Versioning Flow. This split provides a focused navigation target for those operational and behavioral views.

## Extraction Rationale
API standards, resource catalog content, and security testing already have separate child files. Rule catalogs and Mermaid flows remain a distinct review concern because they connect request behavior, error behavior, and compatibility expectations across the API surface.

## Preservation Rule
Do not remove or rewrite the parent API Governance Framework body from this split. This child document is a navigation and ownership slice for the existing canonical content.
