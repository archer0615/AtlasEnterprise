# API Governance Security Testing and Operations

## Purpose
This split document isolates API authentication, authorization, audit, tracing, performance, caching, rate limiting, OpenAPI, validation, business rules, testing, and edge cases from the parent API Governance Framework.

## Source
- Parent specification: [API Governance Framework](../api-governance-framework.md)

## Security
API security covers authentication, authorization, permission mapping, audit, correlation identifiers, causation identifiers, tracing, security headers, and field-level protection.

## Operations
Operational standards cover performance, caching, rate limiting, OpenAPI convention, JSON naming, DateTime convention, money convention, decimal precision, and enum serialization.

## Testing
Testing covers API contract, OpenAPI, security, performance, compatibility, error handling, validation, and edge cases.
