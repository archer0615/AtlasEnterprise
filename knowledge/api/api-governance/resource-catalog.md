# API Governance Resource Catalog

## Purpose
This split document isolates the complete API resource catalog from the parent API Governance Framework.

## Source
- Parent specification: [API Governance Framework](../api-governance-framework.md)

## Resource Groups
- Users API
- Households API
- Blueprint API
- Goals API
- Portfolios API
- Loans API
- Properties API
- Scenarios API
- Decisions API
- Recommendations API
- Policies API
- Dashboard API
- Notifications API
- Reports API
- Administration API
- Audit API

## Catalog Rules
Each resource group must define route, application service, domain service, aggregate, commands, queries, DTO, repository, domain events, permissions, audit requirement, idempotency requirement, and error catalog references.
