# Atlas KPI Definitions
Version: 1.0

## Purpose
This document defines every Key Performance Indicator (KPI) used by Project Atlas. Each KPI has a unique identifier, formal definition, calculation method, unit, target, update frequency and owning engine.

---

## KPI Metadata Standard

Each KPI includes:
- KPI ID
- Name
- Business Purpose
- Formula
- Inputs
- Unit
- Frequency
- Target
- Warning Threshold
- Critical Threshold
- Source Engine

---

## Cash Flow KPIs

### KPI-CF-001 Monthly Cash Flow
Formula:
Income - Expenses

Unit:
Currency / Month

Target:
Positive

Owner:
Cash Flow Engine

---

### KPI-CF-002 Savings Rate

Formula:
Savings ÷ Income

Unit:
%

Target:
>=20%

---

### KPI-CF-003 Emergency Fund

Formula:
Liquid Assets ÷ Monthly Expenses

Unit:
Months

Target:
6–12 months

---

## Loan KPIs

### KPI-LN-001 Debt-to-Income Ratio

Formula:
Monthly Debt Payments ÷ Monthly Income

Target:
<40%

---

### KPI-LN-002 Loan-to-Value (LTV)

Formula:
Outstanding Loan ÷ Property Value

Target:
Scenario dependent

---

### KPI-LN-003 Interest Coverage

Formula:
Disposable Income ÷ Loan Payment

Higher values indicate stronger repayment capacity.

---

## Investment KPIs

### KPI-IV-001 Portfolio Return

Formula:
(Current Value - Invested Capital) ÷ Invested Capital

---

### KPI-IV-002 Portfolio Volatility

Annualized standard deviation of returns.

---

### KPI-IV-003 Asset Allocation Drift

Formula:
Current Allocation - Target Allocation

Target:
Within policy tolerance.

---

### KPI-IV-004 Dividend Yield

Annual Dividend ÷ Portfolio Value

---

## Retirement KPIs

### KPI-RT-001 Retirement Funding Ratio

Formula:
Current Retirement Assets ÷ Required Retirement Assets

Target:
>=100%

---

### KPI-RT-002 Safe Withdrawal Rate

Formula:
Annual Withdrawal ÷ Retirement Assets

---

### KPI-RT-003 Retirement Success Probability

Derived from simulation.

Target:
>=90%

---

## Housing KPIs

### KPI-HM-001 Housing Affordability

Formula:
Housing Cost ÷ Household Income

---

### KPI-HM-002 Home Equity Ratio

Formula:
Equity ÷ Property Value

---

## Risk KPIs

### KPI-RK-001 Overall Risk Score

Weighted aggregation of all risk dimensions.

Range:
0-100

---

### KPI-RK-002 Liquidity Risk

Based on emergency fund and cash flow.

---

### KPI-RK-003 Leverage Risk

Based on debt metrics.

---

## Decision KPIs

### KPI-DC-001 Financial Health Score

Composite score generated by Decision Engine.

Range:
0-100

---

### KPI-DC-002 Goal Achievement Score

Completed Goals ÷ Planned Goals

---

## Governance

- KPI IDs are immutable.
- Formula revisions require version updates.
- KPI calculations always reference the financial assumption version.
- Historical KPI values are never overwritten.
- KPI formulas are shared across all engines to ensure consistency.

---

## Phase 2 Executable Specification

### KPI Calculation Contract

| Field | Required | Description |
|-------|----------|-------------|
| KPICalculationId | Yes | Stable calculation identifier |
| KPIId | Yes | Canonical KPI identifier |
| HouseholdId | Yes | Household scope |
| Domain | Yes | Cash Flow, Loan, Investment, Retirement, Housing, Risk, Decision |
| FormulaVersion | Yes | Formula version used |
| AssumptionVersionSet | Yes | Assumptions used |
| SourceSnapshotId | Yes | Source data snapshot |
| KPIValue | Yes | Calculated value |
| ThresholdState | Yes | Normal, Watch, Warning, Critical |
| TraceId | Yes | Audit trace |

### Commands

| Command | Actor | Output |
|---------|-------|--------|
| CalculateKPI | KPIApplicationService | KPICalculated |
| PublishKPIDefinition | GovernanceApplicationService | KPIDefinitionPublished |
| EvaluateKPIThreshold | KPIApplicationService | KPIThresholdEvaluated |
| ReplayKPICalculation | AdministrationApplicationService | KPICalculationReplayed |

### Validation Rules

| Rule ID | Rule | Failure |
|---------|------|---------|
| KPI-VR-001 | Every KPI must have immutable ID, formula, unit, owner, and thresholds. | Reject definition |
| KPI-VR-002 | KPI calculation must reference formula version and source snapshot. | Reject calculation |
| KPI-VR-003 | Historical KPI values are append-only. | Reject overwrite |
| KPI-VR-004 | Threshold ranges must not overlap. | Reject definition |
| KPI-VR-005 | Replay must use original formula, assumption, and source versions. | Reject replay |

### API Contract

| Endpoint | Method | Purpose | Permission |
|----------|--------|---------|------------|
| /api/v1/kpis | GET | List KPI definitions | KPI:Read |
| /api/v1/kpis/calculate | POST | Calculate KPI values | KPI:Evaluate |
| /api/v1/kpis/{kpiId}/history | GET | Retrieve KPI history | KPI:Read |
| /api/v1/kpis/{kpiId}/replay | POST | Replay KPI calculation | Audit:Replay |

### Testing Matrix

| Test | Expected Result |
|------|-----------------|
| Missing formula version | Calculation is rejected |
| Overlapping thresholds | Definition publish fails |
| Historical replay | Original KPI value reproduces |
| Threshold breach | Threshold event is emitted |
| Missing owner engine | Definition publish fails |
