# Domain Model Governance

## Purpose
This split document isolates domain model governance, validation, security, audit, migration, and testing from the parent Domain Model Catalog.

## Source
- Parent specification: [Domain Model Catalog](../domain-model-catalog.md)

## Governance
Governance covers naming, aggregate alignment, lifecycle consistency, command/event ownership, persistence boundaries, API ownership, security classification, audit coverage, and migration compatibility.

## Testing
Testing covers entity consistency, relationship ownership, command routing, event routing, repository mapping, API mapping, edge cases, and final catalog verification.
