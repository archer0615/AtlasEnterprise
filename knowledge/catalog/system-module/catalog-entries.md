# System Module Catalog Entries

## Purpose
This split document isolates system module definitions from the parent System Module Catalog.

## Source
- Parent specification: [System Module Catalog](../system-module-catalog.md)

## Module Scope
System modules define the deployable or logical ownership surface for APIs, services, repositories, commands, events, jobs, schedulers, automations, permissions, and documentation.

## Entry Contract
Each module entry preserves module name, purpose, bounded context, owned services, aggregates, APIs, jobs, events, repositories, permissions, dependencies, and governance references.
