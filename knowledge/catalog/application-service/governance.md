# Application Service Governance

## Purpose
This split document isolates application service governance, validation, security, audit, performance, testing, and consistency checks from the parent Application Service Catalog.

## Source
- Parent specification: [Application Service Catalog](../application-service-catalog.md)

## Governance
Governance covers service ownership, API alignment, permissions, transaction scope, idempotency, correlation identifiers, audit emission, cache invalidation, and error handling.

## Testing
Testing covers command orchestration, query projection, permission enforcement, DTO mapping, error handling, audit emission, cache behavior, performance, concurrency, and edge cases.
