# Message Contract Governance and Testing

## Purpose
This split document isolates message contract security, audit, validation, testing, edge cases, and consistency checks from the parent Message Contract Catalog.

## Source
- Parent specification: [Message Contract Catalog](../message-contract-catalog.md)

## Governance
Governance covers ownership, schema validation, permission mapping, field masking, audit metadata, contract lifecycle, compatibility review, and release controls.

## Testing
Testing covers producer contracts, consumer contracts, schema validation, replay, retry, dead letter, compatibility, permission masking, and edge cases.
