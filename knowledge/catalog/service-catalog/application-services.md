# Service Catalog Application Services

## Purpose
This split document isolates application service entries from the parent Service Catalog.

## Source
- Parent specification: [Service Catalog](../service-catalog.md)

## Application Service Scope
Application services orchestrate commands, queries, permissions, DTO mapping, validation coordination, transaction boundaries, audit requests, and integration with domain services.

## Catalog Usage
Each application service entry must preserve service name, owning module, consumed aggregate, commands, queries, permissions, events, repositories, and API surface.
