# Bounded Context Governance

## Purpose
This split document isolates bounded context governance, validation, security, audit, testing, and consistency verification from the parent Bounded Context Catalog.

## Source
- Parent specification: [Bounded Context Catalog](../bounded-context-catalog.md)

## Governance
Governance covers context ownership, data boundaries, permission mapping, audit ownership, transaction boundaries, API ownership, repository access, and integration rules.

## Testing
Testing covers context isolation, integration contracts, command routing, event routing, permission mapping, data ownership, migration safety, and edge cases.
