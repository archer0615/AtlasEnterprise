# Bounded Context Definitions

## Purpose
This split document isolates bounded context definitions from the parent Bounded Context Catalog.

## Source
- Parent specification: [Bounded Context Catalog](../bounded-context-catalog.md)

## Definition Scope
Each bounded context defines domain purpose, ownership, aggregates, entities, application services, domain services, repositories, APIs, messages, permissions, and data ownership.

## Catalog Usage
Bounded context definitions are the source for context boundaries, module ownership, service routing, and cross-context integration controls.
