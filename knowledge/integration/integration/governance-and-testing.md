# Integration Governance and Testing

## Purpose
This split document isolates integration matrices, governance, validation, audit, performance, testing, edge cases, and consistency checks from the parent Integration Framework.

## Source
- Parent specification: [Integration Framework](../integration-framework.md)

## Governance
Governance covers integration matrix, system matrix, module matrix, application-service external matrix, domain-event external matrix, message external matrix, API external matrix, security, audit, and performance.

## Testing
Testing covers contract tests, authentication, authorization, retry, timeout, rate limiting, circuit breaker, webhook callback, polling, mapping, and failure recovery.
