# Integration Resilience and Security

## Purpose
This split document isolates integration authentication, authorization, resilience, retry, timeout, circuit breaker, bulkhead, rate limiting, webhook, callback, and polling strategies from the parent Integration Framework.

## Source
- Parent specification: [Integration Framework](../integration-framework.md)

## Security
Security strategies include API key, OAuth, JWT, mTLS, authorization mapping, credential rotation, tenant isolation, and external data masking.

## Resilience
Resilience strategies include retry, timeout, circuit breaker, bulkhead, rate limiting, webhook verification, callback validation, polling safety, and duplicate handling.
